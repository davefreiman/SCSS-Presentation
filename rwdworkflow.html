Design

1. Grid

12 column is easiest to work with, as it divides nicely going from large screens down to mobile.

Columns can be nested as well.

However, we can do other patterns as well, such as 16 or 24 column layouts.

4. Foundation

Mostly using it for it's grid and the fact it's set-up very well in SCSS. It has quite a few handy default components out-of-the-box, as well as several JS plugins to enhance things. Such as an image slider, gallery, lightbox, etc. These compenents can be easily pulled in or not included without much hassle. SCSS makes the big difference here.

Workflow

  1. If we go the rapid prototyping route, then a designer would work in sprints/iterations with a front-end dev to sort out the UI and reflows. Once that's gone through a few sprints and testing, then start making the PSD.
  2. Otherwise, same as before, but now PSD files should have the correct naming convention for the layers and importing into Adobe Edge Reflow to see how content / UI reflows. Occasionally conferring with FE Dev / UX person for UI elements and reflows that are novel/experimental. Then hand-off to FE Dev as before.
